API Reference — Base URL: {{BASE}}
POST /api/v2/saas/address
Allocate a deposit address for your end user. Returns unique address per externalUserId + chain. Wallet generated via HD derivation — private keys stay secure.
Request
curl -X POST {{BASE}}/api/v2/saas/address \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_KEY" \
  -d '{"externalUserId":"user_001","chain":"sepolia"}'
Response 200
{"address":"0xAbC...123","externalUserId":"user_001","chain":"sepolia","createdAt":"..."}
GET /api/v2/saas/addresses
List all allocated addresses with balances and status. Supports pagination and chain filtering.
curl {{BASE}}/api/v2/saas/addresses?chain=sepolia&limit=50&offset=0 \
  -H "x-api-key: YOUR_KEY"
Response 200
{"items":[{"address":"0x...","externalUserId":"user_001","chain":"sepolia","balance":"1.5","tokenSymbol":"ETH"}],"total":150}
GET /api/v2/saas/balances
Aggregated balance summary across all deposit addresses, grouped by token and chain.
curl {{BASE}}/api/v2/saas/balances \
  -H "x-api-key: YOUR_KEY"
Response 200
{"balances":[{"token":"ETH","chain":"sepolia","total":"12.5","addressCount":45},{"token":"USDT","chain":"sepolia","total":"5000","addressCount":20}]}
POST /api/v2/saas/withdraw
Submit a withdrawal request. Evaluated against your auto/review rules. We execute and callback with result.
curl -X POST {{BASE}}/api/v2/saas/withdraw \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_KEY" \
  -d '{"toAddress":"0x...","token":"USDT","amount":"100","chain":"sepolia","externalUserId":"user_001"}'
Response 202
{"withdrawalId":"wd_...","status":"pending_review","message":"Submitted for review"}
GET /api/v2/saas/withdrawals
Query withdrawal history with status, tx hashes, and callback delivery status.
curl {{BASE}}/api/v2/saas/withdrawals?status=confirmed&limit=20 \
  -H "x-api-key: YOUR_KEY"
Response 200
{"items":[{"id":"wd_...","status":"confirmed","txHash":"0x...","callbackStatus":"delivered","amount":"100","token":"USDT","toAddress":"0x...","createdAt":"..."}],"total":42}
📡 Callback Convention
We POST withdrawal status updates to your callback_url (configured in Withdrawal Rules).
Signature: HMAC-SHA256 of JSON body with your api_secret → X-InfraX-Signature header.
Retry: Up to 5 retries at 10s/30s/1min/5min/15min intervals. Check callbackStatus field in withdrawals API.
Idempotency: Same withdrawal_id may be delivered multiple times — deduplicate on your end. 
POST https://your.app/callback
Headers:
  Content-Type: application/json
  X-InfraX-Signature: sha256=abc123...
  X-InfraX-Event: withdrawal.update
Body:
{
  "event": "withdrawal.update",
  "withdrawal_id": "wd_...",
  "status": "confirmed|failed|approved|rejected",
  "to_address": "0x...",
  "from_address": "0x...",
  "token": "USDT",
  "token_symbol": "USDT",
  "amount": "100",
  "tx_hash": "0x...",
  "fail_reason": null,
  "timestamp": "2026-07-03T12:00:00.000Z"
}
🔐 Authentication
All API requests require x-api-key header. Generate your key above.
All amounts are in the token's smallest unit (e.g. wei for ETH, raw for USDT based on decimals).
Rate limit: 100 requests/minute per key. WebSocket events available in beta — contact us for access.
